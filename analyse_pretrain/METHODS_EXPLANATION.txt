Method 1: Frozen feature evaluation (eval_frozen_features.py)
Question answered:
- Did SSL pretraining improve representation quality before fine-tuning?
- Is final checkpoint better than intermediate checkpoint in separability of disease classes?

Why this method:
- This is a standard SSL evaluation protocol in research.
- It avoids full fine-tuning and directly evaluates learned features.
- kNN tests local neighborhood quality.
- Linear probe tests global linear separability.
- Balanced accuracy + macro-F1 are important for class imbalance in OCT disease data.

Why CSV:
- CSV metadata gives one clean source of truth for image path, split, and label.
- Same split for all checkpoints guarantees fair comparison.

Method 2: Retrieval analysis (method_02_retrieval.py)
Question answered:
- Are nearest neighbors semantically and medically consistent in embedding space?
- Does final checkpoint retrieve more same-disease images than intermediate?

Why this method:
- Strong for interpretability of embedding quality.
- Gives both quantitative metric (recall@k) and qualitative evidence (neighbor examples).
- Useful for thesis figures: query image + retrieved neighbors.

Why CSV:
- Needed to map each image to label and split.
- Ensures retrieval is evaluated on controlled subset (e.g., test only).

Method 3: Attention rollout (method_03_attention_rollout.py)
Question answered:
- Where does the ViT focus spatially on OCT images?
- Does focus move toward meaningful retinal structures with better checkpoints?

Why this method:
- Backbone-level spatial analysis before classifier fine-tuning.
- More directly tied to ViT attention mechanism than Grad-CAM on a classifier head.
- Best used as supporting evidence, not sole proof of explanation.

Why CSV:
- Needed to select images and labels for balanced visual inspection by class.

Method 4: Patch-token PCA (method_04_patch_pca.py)
Question answered:
- How are local patch representations organized in each image?
- Are structures/pathologies represented with clearer spatial patterns in later checkpoints?

Why this method:
- Uses token-level features from the pretrained ViT backbone.
- Good for visualizing local representation geometry.
- Useful for comparing intermediate vs final checkpoint behavior on the same images.

Why CSV:
- Needed for deterministic image selection and class-specific analyses.

Method 5: JSON aggregation (method_05_compare_checkpoints.py)
Question answered:
- How to compare all checkpoint results in one table quickly?

Why this method:
- Keeps analysis reproducible and compact.
- Produces a clean CSV table for thesis plots and reporting.
- Makes multi-checkpoint comparison easy.

Important scientific note:
- Strongest evidence comes from Method 1 + Method 2.
- Method 3 and Method 4 are supportive visual analyses.
- For thesis quality, report quantitative metrics first, then visual evidence.
